XeprAPI module¶

Xepr API module for Python, providing access to Xepr via libxeprapi.so

XeprAPI.getXeprInstances()¶

Detects Xepr instances which are waiting for connections from XeprAPI clients. For this to work, the API has to be enabled in Xepr (menu “Processing” -> sub-menu “XeprAPI” -> menu item “Enable Xepr API”). Xepr instances which already are connected to an XeprAPI client are not listed.

Returns:Dictionary of Xepr process ID numbers as its keys and the visible window title of the Xepr instance as the corresponding values. The process ID (PID) keys can be used to connect to a specific Xepr instance by specifying the PID when creating an object of Xepr.

Xepr class¶

class XeprAPI.Xepr(constantconstants=True, libxeprapi=None, verbose=False, pid=None)¶

Initializes the Xepr object and establishes a connection to the Xepr software. For this to work, the API has to be enabled in Xepr (menu “Processing” -> sub-menu “XeprAPI” -> menu item “Enable Xepr API”).

Parameters:
  • constantconstants (True or False; default = True) – If True, ProDeL constants are only requested once upon API initialization; if False, the constants are requested from Xepr each time they are used in XeprAPI classes or in the Python script instantiating the class object. Typically, there is no reason to change the default behavior.
  • libxeprapi (string or None; default = None) – Specify path to the helper library libxeprapi.so. If no path is given, XeprAPI looks for the library in the directory where the XeprAPI Python module is located as well as in the current directory.
  • verbose (True or False; default = False) – If True, enables a few messages in the XeprAPI module.
  • pid (integer value or None) – Connect to a specific Xepr instance (corresponding to this process ID). If no pid value is specified an Xepr instance will be sought out upon construction.
Returns:

Instance of Xepr

Example:

>>> import XeprAPI

>>> Xepr=XeprAPI.Xepr()

XeprDataset(size=None, autorefresh=False, xeprset="primary", iscomplex=False, shape=None)¶

Create a Dataset object for accessing Xepr’s datasets or creating new datasets (both 1D and 2D).

Parameters:
  • size (integer value or tuple of integer values or None; default = None) – Size of dataset to be created. If size = None, no dataset is created, but it will be retrieved from the Xepr dataset specified by xeprset upon access to one of the attributes of the dataset object. If size is an integer number, a 1D dataset will be created, with size being the number of values on the abscissa. If size is a tuple of two integer numbers, i.e. (X,Y), a 2D dataset will be created, with X being the number of values on the first abscissa and Y the number of values on the second abscissa.
  • shape (integer value or tuple of integer values or None; default = None) – similar to size argument, but in reversed order for 2D datasets, where shape = (Y,X) will create a dataset with X being the number of values on the first abscissa and Y the number of values on the second abscissa. For 1D datasets, size and shape have the same meaning.
  • autorefresh (True or False; default = False) – If True, the GUI of the Xepr application will be refreshed each time the update() method is called. If False, the GUI is refreshed only by calling the XeprGUIrefresh() method or if the update() method is called with the ‘refresh’ option.
  • xeprset (string, default = “primary”) – Select the dataset of the Xepr application to be used for writing a dataset to (by calling update()) or for retrieving a dataset from.
  • iscomplex (True or False; default = False) – If True, a dataset would be created with a complex ordinate. Only valid if size != None, i.e. if a dataset is to be created.

Examples:

# ...suppose we already have the Xepr object...
>>> dset_pri = Xepr.XeprDataset()                          # retrieve Xepr's primary dataset
>>> dset_sec = Xepr.XeprDataset(xeprset = "secondary")     # retrieve Xepr's secondary dataset

>>> dset1D = Xepr.XeprDataset(size = 1024,                 # create 1D dataset with 1024 abscissa values
                              xeprset = "secondary")       # associated with Xepr's secondary dataset

>>> dset2D = Xepr.XeprDataset(size = (1024, 25))           # create 2D dataset with X * Y = 1024 * 25 values
# this produces the same type of dataset:
>>> dset2D = Xepr.XeprDataset(shape = (25, 1024))          # using shape argument: Y * X = 25 * 1024 values
XeprExperiment(name_or_vp=-1, exptype=None, axs1=None, axs2=None, ordaxs=None, addgrad=False, addgonio=False, addvtu=False)¶

Creates Experiment object to create a new experiment or to access an experiment already set up by the operator of the Xepr application.

Parameters:
  • name_or_vp (string or integer number; default = -1) – If name_or_vp is an integer value, the Experiment object accesses the experiment corresponding to the Xepr viewport specified by name_or_vp; by default, the viewport number is -1, i.e. the current viewport. If name_or_vp is a string, it is used as the name of the experiment in Xepr to be created or accessed. If none of the remaining parameters is explicitly set, the Experiment object accesses the Xepr experiment by the name given for name_or_vp. If in addition to the string in name_or_vp any parameter is given, Xepr will try to build the experiment described by the parameters.
  • exptype (string or None; default = None) – type of the experiment to be built, e.g. “C.W.” or “Pulse”.
  • axs1 (string or None; default = None) – Type of the first abscissa, e.g. “Field”; required for building a 1D or 2D experiment.
  • axs2 (string or None; default = None) – Type of the first abscissa, e.g. “Field”; required for building a 2D experiment.
  • ordaxs (string or None: default = None) – Type of the ordinate, e.g. “Signal channel” or “Transient recorder”; required for building 1D or 2D experiments.
  • addgrad – If True, add gradient unit to the experiment to be built.
  • addgonio – If True, add goniometer unit to the experiment to be built.
  • addvtu – If True, add temperature controller to the experiment to be built.

Examples:

# ...suppose we already have the Xepr object...
>>> cur_exp = Xepr.XeprExperiment()     # access current experiment in Xepr

>>> cw_exp = Xepr.XeprExperiment("CWExp", exptype="C.W.", axs1="Field",   # create CW experiment
                                 ordaxs="Signal channel", addgrad=True)   # and add a gradient unit

>>> pulse_exp = Xepr.XeprExperiment("PulseExp", exptype="Pulse",                # create a pulse
                                   axs1="Field", ordaxs="Transient recorder")   # experiment
XeprActive()¶
Returns:True if the Xepr API is active.
XeprClose()¶

Closes the API and tells Xepr to shut down its API support as well. After that, the API support of Xepr has to be re-enabled manually by the Xepr operator (menu “Processing” -> sub-menu “XeprAPI” -> menu item “Enable Xepr API”) to be able to establish the Xepr API again.

Raises :IOError exception in case shutting down the Xepr API did not succeed.
XeprGUIrefresh()¶

Refreshes the Xepr GUI.

XeprOpen()¶

Opens the Xepr API for Python. Can be used to re-open the API after having closed it using XeprClose(). Usually, you will not need to open the API manually, since it will be opened automatically upon instantiation of the Xepr class, i.e. by creating an instance of XeprAPI.Xepr().

Raises :IOError exception in case opening the Xepr API did not succeed.

Dataset class¶

class XeprAPI.Dataset(object)¶
X¶

1D (Numpy) array corresponding to the X abscissa values of the dataset

Y¶

1D (Numpy) array corresponding to the Y abscissa values of the dataset (available for 2D datasets only)

O¶

1D or 2D (Numpy) array corresponding to the ordinate values of the dataset

Example:

xabscissa = dset.X              # get X abscissa values
ordinate = dset.O               # get ordinate data
dset.O = dset.O / max(ordinate) # normalize ordinate values
isComplex¶

True if Dataset instance contains complex ordinate data.

shape¶

Shape of the dataset, i.e. tuple of one or two integer values corresponding to the number of values of the (second and) first abscissa.

size¶

Tuple equivalent to shape attribute, but in reversed order. That is, the first integer value corresponds to the number of values on the first abscissa, the second value corresponds to the number of values on the second abscissa (if any).

fromXepr(xeprset=None)¶

Update the dataset in the Dataset instance with data from Xepr. Calls update() with the reverse parameter set accordingly. See update() for a description of the remaining parameter(s).

Examples:

# ...suppose we already have the Xepr object...
>>> exp = Xepr.XeprExperiment()                         # get object to control current experiment in Xepr
>>> exp.aqExpRunAndWait()                               # run the experiment once
>>> dset = Xepr.XeprDataset()                           # retrieve Xepr's primary dataset
>>> print max(dset.O)                                   # print maximum ordinate value
>>> exp.aqExpRunAndWait()                               # run the experiment again
>>> dset.fromXepr()                                     # update the Dataset instance with the new data
>>> print max(dset.O)                                   # print maximum ordinate value for this run

toXepr(refresh=False, xeprset=None, store=False)¶

Update the dataset in Xepr with the data in the Dataset instance. Calls update() with the reverse parameter set accordingly. See update() for a description of the remaining parameters.

Examples:

# ...suppose we already have the Xepr object...
>>> dset_sec = Xepr.XeprDataset(xeprset = "secondary")  # retrieve Xepr's secondary dataset
>>> dset_sec.O -= dset_sec.O.sum() / len(dset_sec.O)    # subtract DC signal
>>> dset_sec.toXepr(refresh=True)                       # update the secondary dataset in the Xepr application
                                                        # and refresh the GUI
update(reverse=False, refresh=False, xeprset=None, store=False)¶

Update the dataset in Xepr with the data in the Dataset instance (if the dataset has been created upon instantiation using the size parameter) or update the Dataset instance with data originating from the corresponding dataset in Xepr (if the Dataset has been instantiated without the size parameter).

Parameters:
  • reverse (True or False; default = False) – If False, the update takes place as described above. If True, the direction of the update is reversed.
  • refresh (True or False; default = False) – If True, the GUI of the Xepr application is refreshed after updating the dataset. Useful if the dataset in the Xepr application is to be updated from the data in the Dataset, since only after refreshing the Xepr GUI the changes will be displayed.
  • xeprset (string or None; default=None) – If not None, the dataset of the Xepr application the Dataset has been associated with upon instantiation will be changed prior to updating. Same as calling the setXeprSet() method before calling update()
  • store (True or False; default = False) – If True, the dataset content will be stored in xepr memory and can be accessed from the drop-down menu of the viewport. The title of the dataset will be used as the designator of the menu entry.

Examples:

# ...suppose we already have the Xepr object...
>>> dset_sec = Xepr.XeprDataset(xeprset = "secondary")  # retrieve Xepr's secondary dataset
>>> dset_sec.O -= dset_sec.O.sum() / len(dset_sec.O)    # subtract DC signal
>>> dset_sec.update(reverse=True,                       # update the secondary dataset in the Xepr application
                    refresh=True)                       # and refresh the GUI

>>> exp = Xepr.XeprExperiment()                         # get object to control current experiment in Xepr
>>> exp.aqExpRunAndWait()                               # run the experiment once
>>> dset = Xepr.XeprDataset()                           # retrieve Xepr's primary dataset
>>> print max(dset.O)                                   # print maximum ordinate value
>>> exp.aqExpRunAndWait()                               # run the experiment again
>>> dset.update()                                       # update the Dataset instance with the new data
>>> print max(dset.O)                                   # print maximum ordinate value for this run
